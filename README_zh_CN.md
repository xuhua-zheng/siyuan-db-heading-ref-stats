# 标题引用同步

把数据库条目绑定文档中的“标题下块引用”，同步到数据库同名文本字段。

简单说：数据库里有一个文本字段叫 `相关人物`，文档里也有一个标题叫 `1.2 相关人物`。在这个标题下面放块引用后，右键数据库条目执行同步，字段 `相关人物` 就会写入这些块引用的名称。

插件只读取已有块引用并写入数据库文本字段，不复制内容，不创建新块，不修改正文，也不会自动创建数据库字段。

## 主要功能

- 数据库条目右键同步：`插件 > 同步标题引用`。
- 多选数据库条目后右键：`插件 > 同步所选标题引用`。
- 数据库文本字段名就是匹配规则：字段名匹配绑定文档中的同名标题。
- 支持任意标题级别，H1 到 H6 都可以。
- 匹配标题时会忽略开头序号，例如 `1.2 相关人物` 可以匹配字段 `相关人物`。
- 默认按被引用块 ID 去重，同一个块被重复引用时只写一次。

## 使用方法

1. 在数据库中创建文本字段，例如 `相关人物`、`引用文献`、`关联项目`。
2. 将数据库条目绑定到对应文档。
3. 在文档中创建同名标题，例如 `1.2 相关人物`。
4. 在该标题下面插入块引用。
5. 回到数据库，右键条目，选择 `插件 > 同步标题引用`。

同步后，插件会把标题下的块引用名称用分隔符连接后写回同名文本字段。

## 同步规则

- 只有数据库条目右键菜单会触发同步。
- 插件不会监听文档变化，不做后台自动同步。
- 插件只处理数据库中的文本字段，其他字段类型不会写入。
- 没有匹配到标题的文本字段保持原值，不会被清空。
- 匹配到标题但标题下没有块引用时，该字段会写为空。
- 同一文档存在多个同名标题时，会合并这些标题下的块引用。
- 插件通过思源公开 API 写入属性视图，不依赖数据库页面是否打开。

## 设置

- 名字分隔符：多个块引用名称之间的分隔符，默认 `、`。
- 按被引用块去重：默认开启。
